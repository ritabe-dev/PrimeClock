# README Surface Policy

The root `README.md` is the public-facing GitHub landing page. Keep it focused
on citable public release information only.

## Root README

The root README may contain:

- current public theorem release metadata;
- foundational public release metadata;
- finite claim boundaries and non-claims;
- verification commands for public release artifacts;
- public release bundle and citation guidance.

The root README must not contain:

- active research-line planning;
- source-only bridge or future-version roadmap details;
- Gate R / Gate C / Gate P workflow notes;
- development app or optional tooling notes;
- DOI finalization or release-registry operating procedure.

## Internal Documentation

Internal release and version-line details belong in:

- `VERSION_MAP.md`;
- `release/public/PUBLISH_CHECKLIST.md`;
- `release/public/MAINTENANCE_POLICY.md`;
- release-specific manifests, workflows, notes, and registry entries.

Future source-only research lines should stay local-first until they are ready
for a deliberate public release or checkpoint.

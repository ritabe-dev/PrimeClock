## Recommendation

The strongest mathematical theme is:

```text
Prime-prefix residue covering filtration.
```

This is stronger than treating `C0(N)=1` as a raw event. It turns PRC complete
coverage into a finite sequence of exact covering problems on residue rings.

## Definition

Let

```text
p_1=2, p_2=3, ..., p_k
M_k = product_{i<=k} p_i
```

For a residue `r mod M_k`, define the prefix arc union

```text
U_k(r) = union_{i<=k} I_{p_i}(r)
```

where

```text
I_p(r) = [(r mod p)/p - 1/(2p), (r mod p)/p + 1/(2p)] on R/Z
```

For theorem-level coverage these are **closed arcs**. This matters at endpoint
touching; for example the `r=2 mod 210` coverage chain touches at `1/2`.
Endpoint convention does not change measures, but it does change whether a
finite exact covering statement is literally true.

Then define the covered residue set

```text
C_k = {r in Z/M_kZ : U_k(r) = R/Z}.
```

This is finite and exact. It has no floating-point component and does not
require a large scan over `N`.

Strictly speaking, the `C_k` are not nested subsets of one fixed finite set.
They form a lifted filtration over the primorial inverse system. With

```text
pi_{k,k-1}: Z/M_kZ -> Z/M_{k-1}Z
Lift_k(C_{k-1}) = pi_{k,k-1}^{-1}(C_{k-1})
B_k = C_k \ Lift_k(C_{k-1})
```

the inherited classes are the cylinder lifts and `B_k` is the exact birth layer.

## Basic Lemmas

### Lemma 1: Prefix Coverage Depends Only on `N mod M_k`

For the first `k` primes, each center is

```text
{N/p_i} = (N mod p_i) / p_i.
```

Therefore the entire prefix union `U_k(N)` depends only on the residue class of
`N mod M_k`.

This means `C_k` is the exact complete-covering set for the prefix system.

### Lemma 2: Lift Monotonicity

If `r in C_k`, then every lift

```text
r + t M_k mod M_{k+1},  t=0,...,p_{k+1}-1
```

belongs to `C_{k+1}`.

Reason: the first `k` prime arcs already cover the circle, and adding the
`p_{k+1}` arc cannot uncover anything.

Thus `C_{k+1}` decomposes into:

```text
inherited lifts from C_k
new births created by p_{k+1}
```

This gives a genuine lifted filtration, not just a list of examples.

### Lemma 3: Density Monotonicity

Let

```text
alpha_k = |C_k| / M_k.
```

Lift monotonicity gives

```text
|Lift_k(C_{k-1})| = p_k |C_{k-1}|
alpha_k = alpha_{k-1} + |B_k| / M_k.
```

Therefore `alpha_k` is nondecreasing. Questions about later decay are not
well-posed for this exact filtration; the meaningful questions are whether
births eventually stop, and how large the monotone limit becomes.

## First Hand-Proof Example

At `k=4`, the first covered residues are:

```text
C_4 = {2, 208} mod 210.
```

For `r=2`, the four arcs from `2,3,5,7` are:

```text
p=2, r mod 2 = 0:
  [0, 1/4] union [3/4, 1]

p=3, r mod 3 = 2:
  [1/2, 5/6]

p=5, r mod 5 = 2:
  [3/10, 1/2]

p=7, r mod 7 = 2:
  [3/14, 5/14]
```

These intervals chain together:

```text
[0,1/4] overlaps [3/14,5/14]
[3/14,5/14] overlaps [3/10,1/2]
[3/10,1/2] touches [1/2,5/6]
[1/2,5/6] overlaps [3/4,1]
```

So the union is all of `R/Z`.

The residue `208 == -2 mod 210` is the mirror case.

This is an exact hand coverage example for the two covered residues. The
statement `C_4={2,208} mod 210` is now supported by the v1.6 row-level
exclusion witnesses, the v1.5 36-class summary table, the v1.7 package
verifier, the v1.8 standard-library standalone checker, and the human-readable
C4 appendix in the finite note. The fully handwritten 208-case compression is
still optional polish, not a blocker for the current finite certificate
package.

## Why This Is Mathematically Promising

This theme has several advantages over the previous experimental themes.

First, it gives exact finite objects:

```text
C_k subset Z/M_kZ
birth set B_k = C_k \\ lifts(C_{k-1})
```

Second, it has natural theorem targets:

- characterize `C_k` for small `k`;
- describe or bound `|C_k| / M_k`;
- characterize birth residues `B_k`;
- understand whether densities stabilize, grow, or fluctuate;
- connect certificate depth of `C0(N)` to the least `k` such that
  `N mod M_k in C_k`.

Third, it clarifies previous selected-window `C0` results. In the checked
certificate table, many complete values are not deep large-`N` events; they
are explained by shallow membership in some `C_k`.

## Relation to Certificate Depth

For a complete-covering `N` with a prefix certificate, define:

```text
k_cert(N) = min{k : N mod M_k in C_k}
P_cert(N) = p_{k_cert(N)}
```

This turns complete covering into a prefix-certificate problem.
If no prefix certificate is found in the checked range, treat `k_cert(N)` as
unknown rather than finite.

The selected-window certificate table already showed:

```text
unique complete values checked: 208
all certificate prime median: 11
nontrivial certificate prime median: 17
max certificate prime: 331
```

The filtration `C_k` gives the theoretical structure behind those empirical
certificate depths.

## Research Program

### Step 1: Exact Small-`k` Classification

Prove:

```text
C_1=C_2=C_3=empty
C_4={2,208} mod 210
```

Then generate and inspect exact symbolic descriptions of `C_5` and `C_6`.

### Step 2: Birth Mechanism

For each new prime `q=p_{k+1}`, study:

```text
B_{k+1} = C_{k+1} \\ lifts(C_k).
```

A residue is born when the old uncovered gaps under `U_k(r)` are all swallowed
by the single new `q`-arc.

Measured birth rows:

```text
k=5, q=11: births=14
k=6, q=13: births=42
k=7, q=17: births=714
```

This suggests a finite combinatorial process on residual gaps.

### Step 3: Density and Growth

Track:

```text
alpha_k = |C_k| / M_k
beta_k  = |B_k| / M_k
```

The first values of `alpha_k` after coverage begins are:

```text
alpha_4 ~= 0.00952
alpha_5 ~= 0.01558
alpha_6 ~= 0.01698
alpha_7 ~= 0.01838
```

Questions:

- Does `alpha_k` approach a positive limit?
- Do births continue infinitely often, or does the lifted filtration eventually
  become purely inherited?
- Is the observed additional growth only an early-prefix artifact?
- Can `alpha_k` be bounded using total arc length and gap combinatorics?

### Step 4: Nontrivial `C0`

Instead of asking whether `C0(N)=1`, ask:

```text
what is k_cert(N)?
```

Then separate:

- shallow deterministic classes;
- medium-depth finite certificates;
- genuinely deep cases where `k_cert(N)` is large.

This is likely a better mathematical classification than complete vs
non-complete alone.

## Comparison to Earlier Themes

This theme subsumes or organizes several previous themes:

- **small-prime wheel strata** become `C_4`;
- **certificate depth** becomes first-entry time into `C_k`;
- **complete-covering cliffs** become local changes in `k_cert` and residual
  gap geometry;
- **adjacent anti-persistence** should be recomputed after stratifying by
  `k_cert`;
- **branch-uniform null** remains a separate geometric comparison for residual
  fragmentation, but is no longer the cleanest exact `C0` framework.

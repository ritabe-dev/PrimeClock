# Prime-prefix residue certificates for a finite circle-covering problem

This note is the v1.1 pivot target for PRC. It reframes the project around an
exact finite object rather than selected modulo-210 diagnostics.

## Definition

Let

```text
p_1=2, p_2=3, ...
M_k = prod_{i<=k} p_i.
```

For a residue `r in Z/M_kZ`, define the closed circle arc

```text
I_p(r) = [(r mod p)/p - 1/(2p), (r mod p)/p + 1/(2p)] in R/Z.
```

The closed-arc convention is part of the theorem-level definition. Measures do
not depend on endpoint choices, but exact coverage can: the base example
`r=2 mod 210` uses endpoint touching at `1/2`.

Define

```text
U_k(r) = union_{i<=k} I_{p_i}(r)
C_k = {r in Z/M_kZ : U_k(r) = R/Z}.
```

The sets `C_k` live in different residue rings, so the word filtration means a
lifted filtration over the primorial inverse system. Let

```text
pi_{k,k-1}: Z/M_kZ -> Z/M_{k-1}Z
Lift_k(C_{k-1}) = pi_{k,k-1}^{-1}(C_{k-1})
B_k = C_k \ Lift_k(C_{k-1}).
```

Here `B_k` is the birth layer: classes not previously covered by the first
`k-1` primes, but covered after adding `p_k`.

## Exact lemmas

**CRT dependence.** For each prefix prime,

```text
{N/p_i} = (N mod p_i) / p_i.
```

Therefore prefix coverage depends only on `N mod M_k`.

**Lift monotonicity.** If `r in C_k`, every lift of `r` to `Z/M_{k+1}Z` is in
`C_{k+1}`. The first `k` arcs already cover the circle, and adding one more
arc cannot uncover it.

**Density monotonicity.** With `alpha_k=|C_k|/M_k`,

```text
alpha_k = alpha_{k-1} + |B_k| / M_k.
```

Thus `alpha_k` is nondecreasing. The right growth question is not whether
`alpha_k` decays, but whether births eventually stop and how large the monotone
limit becomes.

**Reflection symmetry.** If `r in C_k`, then `-r mod M_k in C_k`. Reflection
of the circle maps each selected residue cell to the reflected selected cell.

## First nonempty layer

The first nonempty layer is

```text
C_1=C_2=C_3=empty
C_4={2,208} mod 210.
```

For `r=2`, the four arcs are:

```text
p=2: [0,1/4] union [3/4,1]
p=3: [1/2,5/6]
p=5: [3/10,1/2]
p=7: [3/14,5/14]
```

They chain across the circle:

```text
[0,1/4] overlaps [3/14,5/14]
[3/14,5/14] overlaps [3/10,1/2]
[3/10,1/2] touches [1/2,5/6]
[1/2,5/6] overlaps [3/4,1]
```

The residue `208=-2 mod 210` is the reflection partner. The exclusion of the
other 208 residues is certified by exact finite enumeration with one rational
gap witness per residue:

```bash
cd research
python -m prime_reciprocal_projection.cli covering-prime-prefix-exclusion-witnesses \
  --k 4 \
  --out data/summaries/prc_prime_prefix_c4_exclusion_witness_v1_2.csv
```

The witness CSV has `208` data rows. Each row gives a nonempty rational
uncovered interval for a residue not in `C_4`. A compressed handwritten
exclusion table remains a polishing target, but the finite certificate artifact
is now auditable.

The 208 witnesses can be compressed into 36 measure/component classes:

```bash
cd research
python -m prime_reciprocal_projection.cli covering-prime-prefix-exclusion-summary \
  --k 4 \
  --out data/summaries/prc_prime_prefix_c4_exclusion_summary_v1_3.csv
```

This summary is the working index for a short written exclusion proof. It
groups residues by uncovered component count and exact uncovered measure, while
retaining residue samples and representative gap intervals.

## Relation to PRC complete covering

For an original PRC integer `N`, a prefix certificate is sufficient:

```text
if N >= p_k and N mod M_k in C_k, then U_N covers the circle.
```

This is not necessary unless the checked prefix includes all primes up to `N`.
Rows without a certificate through checked `k` should be called
`uncertified within max_k`, not non-covered.

## Non-claims

- This is not a new limiting distribution for `{N/p}`.
- This is not a theorem about the asymptotic behavior of `|C_k|/M_k`.
- This does not say `C_4` or `C_5` explains all complete PRC events.
- This does not make selected modulo-210 diagnostics representative.
- This does not use floating-point coverage checks for theorem-level claims.

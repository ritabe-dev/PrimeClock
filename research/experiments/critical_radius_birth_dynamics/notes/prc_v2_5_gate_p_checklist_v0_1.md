# PRC v2.5 Gate P Checklist

Status: candidate-included checklist for the next public-decision slice.  This
note does not authorize a public release.

## Purpose

Gate P decides whether the v2.5 candidate is stable enough to become a public
claim and release artifact.  Gate P is separate from Gate R research selection
and Gate C candidate-package reproducibility.

## Checklist

- claim wording stable
- B8 boundary stable
- external reviewer can reproduce Gate C
- public README non-claims clear
- release metadata ready
- Zenodo DOI plan ready
- no B8 full graph
- no B8 public theorem
- avoid general predictor wording
- describe the result as a finite exact aperture-orbit separator / certificate
- phase margin is an exact signed containment certificate, not a general predictor
- B8 selected stress-control sample is not coverage, recall, or holdout validation
- obstruction buckets are finite diagnostics, not a layer-general causal taxonomy

## Required Boundary

B8 remains a selected stress-control sample, not coverage, recall, or holdout
validation.  The checked B8 counts are stress-control counts, not public
evidence.  They are useful for stress testing the v2.5 separator, but they do
not establish a B8 full graph claim, a public B8 theorem, an asymptotic law, or
a general prediction theorem.

Obstruction bucket counts are finite diagnostics for the checked scopes.  They
must not be stated as a layer-general causal taxonomy.

## Gate P Default

The default Gate P decision remains `not ready` until the public-facing claim
text, release notes, public README non-claims, release metadata, and DOI plan
are checked together in a later slice.
